# 获取四六级真题网页 PPT 设计稿

- 日期：2026-05-20
- 主题：获取四六级真题，不要再在评论区刷 ×× 啦！
- 产物类型：单文件 HTML 横向翻页网页 PPT
- 设计基底：`guizang-ppt-skill` 风格 A `电子杂志 × 电子墨水`

## 1. 项目目标

将现有文章《获取四六级真题，不要再在评论区刷 ×× 啦！》改写为一份适合互联网公开传播的轻量型网页 PPT。

这份 deck 不是演讲稿，也不是教程文档，而是一份面向公众的介绍型内容，目标是在较短篇幅内完成三件事：

1. 让用户记住“有一个免费、无广、免登录的四六级真题网站”。
2. 让用户愿意把这份内容转发给有需要的人。
3. 在完成信息传达后，自然地承接作者关注。

## 2. 受众与传播场景

- 核心受众：需要找四六级真题的互联网公众
- 更聚焦的人群：临近四六级考试、正在找真题和听力资源的用户
- 传播场景：多平台发布，不绑定某一个平台的导流逻辑
- 传播特征：用户会快速滑读，停留时间有限，因此页数必须短、信息密度必须高、标题必须抓人

## 4. 风格与语气

### 4.1 视觉方向

- 使用 `guizang-ppt-skill` 风格 A：`电子杂志 × 电子墨水`
- 主题色：`🖋 墨水经典`
- 视觉目标：像一篇被认真编辑过的杂志式网页内容，而不是“截图拼贴广告页”

选择这套方向的原因：

- 用户目标是公开传播，不是技术汇报，杂志感更容易被公众接受
- 现有截图本身界面较干净，适合轻包装、保真展示
- 文章原始气质偏“个人观察 + 实用分享”，与风格 A 更一致

### 4.2 文案语气

- 主语气：`吐槽 + 实用分享`
- 节制原则：
  - 吐槽只负责抛出痛点，不扩展成情绪宣泄
  - 正文保持简短，像精修过的网文，不像说明书
  - 重点始终回到“我已经做了一个更直接的方案”

### 4.3 CTA 语气

结尾固定使用：

`如果这份资源对你有帮助，欢迎关注我`

不补“在你看到的平台关注我”之类的说明，不做平台限定，不写多余解释。

## 5. 叙事策略

采用 `痛点反击型` 结构，而不是单纯的工具介绍型或作者动机型。

核心叙事主线：

`别再刷评论区了` → `现有获取方式很烦` → `我做了一个更直接的解法` → `它真的能用` → `如果有帮助，欢迎关注我`

选择该结构的原因：

- 更适合互联网公开传播
- 标题抓人，容易形成停留
- 能同时承接情绪与信息，不会像广告，也不会像教程
- 与原文“我不喜欢这种中介形式，所以我去做了一个网站”的逻辑完全一致

## 6. 页数与节奏

- 总页数：`8 页`
- 信息策略：短 deck、快滑读、强标题、少解释
- 不做过长信息铺陈，不追求覆盖所有细节

### 6.1 主题节奏

按照风格 A 的主题呼吸规则，固定为：

1. `hero dark`
2. `hero light`
3. `light`
4. `dark`
5. `light`
6. `dark`
7. `light`
8. `hero light`

该节奏满足：

- 有开场冲击
- 有中段明暗切换
- 有至少两个 hero 页制造呼吸
- 不会连续 3 页以上同主题

## 8. 截图处理策略

用户已明确选择：

`轻包装但保真`

这意味着：

- 不重画网页
- 不改原始界面结构
- 不做夸张营销风卡片
- 只做统一留白、纸感背景、轻边框、轻阴影、版心对齐

### 8.1 处理原则

- 优先保留界面真实信息
- 包装只服务于版式统一，不服务于“变好看但失真”
- 截图看起来应像“同一篇杂志里的图版”，而不是零散拼接的系统截图

## 10. 实现边界

本设计稿阶段只确认：

- 叙事结构
- 页数
- 版式分配
- 截图策略
- 文案语气
- CTA 方向

本阶段不执行：

- HTML 实现
- 图片重命名
- 截图合成与美化
- 页面开发与调试

这些工作将在设计稿确认后进入单独的实现计划。

## 11. 成功标准

如果最终成品满足以下条件，则视为设计目标达成：

1. 用户能在很短时间内理解你在解决什么问题。
2. 用户能记住“免费、无广、免登录”这三个核心特征。
3. 用户能通过截图判断网站是真实可用的。
4. 用户不会觉得这是一页页功能说明书或硬广告。
5. 结尾 CTA 自然，不突兀，不平台绑定。
